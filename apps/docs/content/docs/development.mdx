# Development

Use `pnpm install`, `just quick`, and `just ci` for local validation. OpenSpec validation is
available through `just openspec`.

Automated tests use local fixtures and local harness pages, not live LinkedIn accounts.

## Inner loop

```bash
pnpm install
pnpm check:profile-output
pnpm --filter @linkedin-profile-exporter/extension typecheck
pnpm --filter @linkedin-profile-exporter/docs build
```

## Profile-output assurance

`pnpm check:profile-output` validates the synthetic Voyager fixture, applies default settings,
asserts that diagnostics/provenance/confidence are stripped, applies `includeAllFields`, and exports
all supported formats.

To validate a local profile JSON file without committing it:

```bash
pnpm check:profile-output /absolute/path/to/profile.json
```

The command prints schema version, section counts, diagnostic codes, and output byte sizes. It does
not print names, profile URLs, role descriptions, or raw profile sections.

## Runtime debugging

| Symptom                               | Check                                                                            |
| ------------------------------------- | -------------------------------------------------------------------------------- |
| Toolbar action inactive               | Confirm the active tab URL matches `https://www.linkedin.com/in/...`.            |
| Popup says content script unavailable | Reload the LinkedIn profile tab, then reopen the popup.                          |
| Extraction is slow                    | Watch the phase label; Voyager attempts are timeout-bounded before DOM fallback. |
| Sections are missing                  | Enable Include all fields, extract again, and inspect diagnostics inventory.     |
