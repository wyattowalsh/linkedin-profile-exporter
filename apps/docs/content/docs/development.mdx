# Development

Use `pnpm install`, `just quick`, and `just ci` for local validation. OpenSpec validation is
available through `just openspec`.

Automated tests use local fixtures and local harness pages, not live LinkedIn accounts.

## Inner loop

```bash
pnpm install
pnpm check:profile-output
pnpm --filter @linkedin-profile-exporter/extension typecheck
pnpm --filter @linkedin-profile-exporter/docs build
```

## Profile-output assurance

`pnpm check:profile-output` validates the synthetic Voyager fixture, applies default settings,
asserts that diagnostics/provenance/confidence are stripped, applies `includeAllFields`, and exports
all supported formats.

To validate a local profile JSON, XML, pasted XML text, or pasted Markdown text file without
committing it:

```bash
pnpm check:profile-output /absolute/path/to/profile.json
pnpm check:profile-output /absolute/path/to/profile.xml
pnpm check:profile-output /absolute/path/to/pasted-text.txt
```

The command prints schema version, section counts, diagnostic codes, warning categories, and output
byte sizes. It does not print names, profile URLs, role descriptions, or raw profile sections.

## Runtime debugging

| Symptom                              | Check                                                                                                                                                                                                                                                                                                             |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Toolbar action unbadged              | Confirm the active tab URL matches `https://www.linkedin.com/in/...`; off-profile tabs stay clickable but unbadged.                                                                                                                                                                                               |
| Popup cannot recover the profile tab | Confirm the generated target supports runtime injection and the manifest lists the LinkedIn content script.                                                                                                                                                                                                       |
| Extraction is slow                   | Watch the phase label; Voyager, detail-page, temporary detail-render, and pagination attempts are timeout-bounded before DOM fallback.                                                                                                                                                                            |
| Sections are missing                 | Enable Include all fields, extract again, and inspect `coverage.<section>.*` diagnostics plus checker default-page-size warnings. Markdown exports include a compact Coverage Diagnostics section only when troubleshooting fields are retained. Enable Verbose diagnostics only when inventory detail is needed. |
