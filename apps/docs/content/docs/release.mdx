# Release

Release readiness requires OpenSpec validation, lint, typecheck, tests, docs checks, asset checks, store material checks, extension builds, manifest checks, and final git whitespace/status audit.

## GitHub Release packet

GitHub Releases are the canonical public artifact handoff for v0.1.0. Build and validate the repo first, then package draft-release assets:

```bash
mise exec node@24 -- just ci
mise exec node@24 -- pnpm release:github:package
```

The package command writes browser ZIPs, the bookmarklet package, a Firefox source-review ZIP, checksums, release notes, and a draft `gh release create` command under `.release/github/v0.1.0/`. Review the generated notes and assets before running the generated command.

The generated `gh release create` command uses `--draft` and `--verify-tag`; the release tag must already exist on the remote so the command cannot create a release from an unintended commit.

Credentialed web-store submissions are not part of v0.1.0 unless explicitly requested later. Keep Chrome Web Store, Microsoft Edge Add-ons, Firefox AMO, and Safari/App Store Connect submission steps separate from the GitHub Release packet.
